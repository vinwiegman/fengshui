# Roomshift — Furniture Layout Optimizer

Scan a room, correct the detected layout, choose an objective, and receive feasible
furniture arrangements generated by a spatial optimization system.

This repository contains the **web client + layout engine** — the interactive half of the
scan-to-layout pipeline. Rooms are loaded from the shared JSON schema
([schemas/room-schema.json](schemas/room-schema.json)), which is the contract the iOS
RoomPlan scanner exports into. Until the scanner lands, three sample rooms
(bedroom, home office, L-shaped living room) stand in for real scans, and any room JSON
can be imported from the top bar.

![Screens: editable floor plan → objectives → generated layouts with explanations]

## What it does

1. **Edit room** — an editable top-down floor plan rendered in SVG. Drag furniture,
   rotate (R), resize, recategorize, delete, add pieces, and mark items *movable* or
   *fixed*. Constraint violations (overlaps, out-of-bounds, blocked doors) are flagged
   live while you drag, and a layout score updates in real time.
2. **Optimize** — pick one or more objectives (*minimal changes*, *maximum open space*,
   *best functional layout*). A simulated-annealing optimizer searches over positions and
   90° rotations of every movable item. Hard constraints are never traded away: furniture
   stays inside the room, never overlaps, and never blocks the door — infeasible proposals
   are rejected outright.
3. **Results** — up to three alternative layouts, each with a mini floor plan, per-term
   score bars, movement arrows over ghosted "before" positions, and a plain-language
   explanation derived from *measured* layout differences (not hallucinated). Apply a
   layout and keep refining it by hand.

## The optimizer

Decision variables: `(x, y, rotation)` per movable item, snapped to 5 cm / 90°.

**Hard constraints** (feasibility, checked geometrically):
- every footprint inside the room polygon (works for non-convex rooms),
- no pairwise overlaps,
- a clearance rectangle in front of every door stays empty,
- fixed items never move.

**Soft objectives** (scored on a 10 cm occupancy grid):

| Term | How it's measured |
| --- | --- |
| Open space | Free-cell ratio, weighted toward the largest *connected* free region (flood fill) |
| Walking paths | BFS from the door over free cells — every key item must be reachable |
| Wall alignment | Large furniture rewarded for having a side against a wall |
| Functional relations | Nightstand↔bed, chair↔desk, sofa↔TV distance rules, desk near window, storage fronts kept clear |
| Movement cost | Total displacement + rotation changes vs. the current layout |

Each objective preset is a weight vector over these terms. Search is simulated annealing
with geometric cooling, Gaussian moves, rotation moves, and occasional teleports to escape
local optima (`src/engine/annealer.js`). The explanation layer (`src/engine/explain.js`)
turns the structured before/after diff into sentences — the same diff contract an LLM can
be swapped into later.

## Run it

```bash
npm install
npm run dev     # http://localhost:5173
```

No backend needed — the engine runs entirely in the browser.

## Project layout

```
src/
├── engine/
│   ├── geometry.js      # footprints, polygon tests, occupancy grid, clearance rects
│   ├── constraints.js   # hard-constraint checker (feasibility)
│   ├── objectives.js    # soft objective terms + presets
│   ├── annealer.js      # simulated annealing search
│   └── explain.js       # measured-diff → plain-language explanations
├── components/          # FloorPlan (SVG editor), Sidebar, Inspector, ResultsPanel
└── data/rooms.js        # sample rooms in the shared schema
schemas/room-schema.json # the scanner ↔ optimizer ↔ client contract
sample-data/             # importable room JSON files
```

## Roadmap

- iOS RoomPlan scanner (Swift) exporting into the shared schema
- CP-SAT solver (OR-Tools) on a discretized grid, benchmarked against annealing
- LLM preference parsing ("more space around the bed") → validated objective weights
- Walking-path *width* scoring, per-room-type rule sets, 3D preview

## Limitations

- Furniture is modeled as oriented rectangles; curved/irregular pieces are approximated.
- Door clearance is a fixed rectangle, not a swing simulation.
- "Better" is only as good as the selected objective weights — that's by design:
  the objective is explicit and measurable, never guessed.
